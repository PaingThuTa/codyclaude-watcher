  <behavior>
    - Hook receives sessionId, toolName, promptText as $1, $2, $3
    - Creates /tmp/codywatcher/sessions/ directory (mkdir -p)
    - Pre-creates FIFO at /tmp/codywatcher/sessions/{sessionId}.fifo if not exists (HOOK-02, D-07)
    - Builds JSON safely with jq (not string interpolation) (HOOK-03, D-08)
    - POSTs to http://localhost:18765/notify with 2-second connection timeout (D-05)
    - If curl fails (daemon down): exits 0 with no stdout output — Claude falls through (HOOK-01 fallback)
    - Blocks on FIFO read with `timeout 30 cat` (HOOK-04, D-09)
    - If FIFO read times out: outputs deny decision JSON with "Daemon timed out" message
    - Echoes only the FIFO decision to stdout (HOOK-05, D-12)
  </behavior>
  <action>
    Create ~/.codywatcher/hook.sh. Set shebang to `#!/bin/bash`. Accept three positional args: SESSION_ID=$1, TOOL=$2, PROMPT=$3.

    Step 1 — FIFO pre-create (HOOK-02, D-07):
    `mkdir -p /tmp/codywatcher/sessions`
    `FIFO="/tmp/codywatcher/sessions/$SESSION_ID.fifo"`
    `[ -p "$FIFO" ] || mkfifo "$FIFO"`

    Step 2 — POST to daemon (HOOK-01, D-05):
    Build JSON with jq: `BODY=$(jq -n --arg sid "$SESSION_ID" --arg tool "$TOOL" --arg prompt "$PROMPT" '{"sessionId":$sid,"tool":$tool,"prompt":$prompt}')`
    curl with 2-second timeout, silence all output: `curl -s --connect-timeout 2 -X POST http://localhost:18765/notify -H "Content-Type: application/json" -d "$BODY" >/dev/null 2>&1`
    If curl exits non-zero: `exit 0` (no stdout — Claude falls through to normal prompt)

    Step 3 — Read decision from FIFO (HOOK-04, D-09):
    `DECISION=$(timeout 30 cat "$FIFO" 2>/dev/null)` || `DECISION='{"hookSpecificOutput":{"hookEventName":"PermissionRequest","decision":{"behavior":"deny","message":"Daemon timed out"}}}'`

    Step 4 — Output decision (HOOK-05, D-12):
    `echo "$DECISION"`

    Make hook.sh executable: `chmod +x ~/.codywatcher/hook.sh`
  </action>
  <verify>
    <automated>bash -n ~/.codywatcher/hook.sh && shellcheck ~/.codywatcher/hook.sh</automated>
    <manual>Start daemon manually, run `~/.codywatcher/hook.sh "test-session" "Bash" "test prompt"` in background, verify daemon receives POST and FIFO contains decision</manual>
  </verify>
  <acceptance_criteria>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| hook.sh → daemon (localhost:18765) | Untrusted prompt text from Claude Code — must be jq-encoded (T-01-01 mitigated) |
| hook.sh stdout → Claude Code | Only decision JSON allowed — curl output must not leak (HOOK-05) |
| /tmp/codywatcher/sessions/ ← hook.sh | FIFO files in shared /tmp — UUID names prevent collision (SEC-02) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Injection | hook.sh JSON encoding | mitigate | jq --arg encoding prevents prompt injection into JSON (HOOK-03, D-08) |
| T-01-07 | Tampering | FIFO decision data | mitigate | Hook reads FIFO once, outputs immediately — no intermediate state to tamper with |
| T-01-08 | Denial of Service | Hook blocking on FIFO | mitigate | 30-second timeout prevents permanent stuck (HOOK-04, D-09) |
| T-01-09 | Information Disclosure | hook.sh stdout leakage | mitigate | curl silenced with >/dev/null 2>&1, only echo "$DECISION" writes stdout (HOOK-05, D-12) |
| T-01-SC | Tampering | npm/pip/cargo installs | mitigate | slopcheck + blocking human checkpoint for [ASSUMED]/[SUS] |
</threat_model>
