  <action>
    Create listen-yesno.swift at ~/.codywatcher/listen-yesno.swift implementing:

    1. Parse --timeout flag (default 10s) from command line arguments
    2. Call SFSpeechRecognizer.requestAuthorization — if not .authorized, print warning and exit 2
    3. Check SFSpeechRecognizer(locale: "en-US").isAvailable — if false, print warning and exit 2
    4. Configure AVAudioSession for recording (.record category, .measurement mode)
    5. Set up AVAudioEngine with input node tap feeding SFSpeechAudioBufferRecognitionRequest
    6. Start recognition task — on each result, check bestTranscription.formattedString.lowercased():
       - If contains "yes" → stop audio engine, cancel task, exit 0
       - If contains "no" → stop audio engine, cancel task, exit 1
    7. Keep run loop alive with RunLoop.main.run(until: Date(timeIntervalSinceNow: timeout))
    8. After timeout: stop audio engine, cancel task, exit 2

    Use the Swift code example from 02-RESEARCH.md as a reference. The binary must:
    - Handle all error cases gracefully (no crashes)
    - Exit cleanly in all paths
    - Use en-US locale for recognition
    - NOT require an app bundle (pure CLI binary)

    After creating the Swift source, compile it:
    cd ~/.codywatcher && swiftc -framework Speech listen-yesno.swift -o listen-yesno

    Verify the binary exists and is executable.

    Behavior expectations:
    - Given a Swift source file with SFSpeechRecognizer setup, when compiled with swiftc -framework Speech, the binary is created at the expected path
    - Given the binary runs with no voice input, when timeout expires (10s), exit code is 2
    - Given the Speech framework is unavailable, when binary runs, exit code is 2
  </behavior>
