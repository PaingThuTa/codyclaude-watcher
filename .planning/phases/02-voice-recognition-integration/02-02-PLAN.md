  <files>~/.codywatcher/daemon.ts, ~/.codywatcher/daemon.test.ts</files>
  <action>
    Add two functions to the existing daemon.ts:

    1. **checkAuthHeader(req: Request, expectedKey: string): boolean**
       - Reads req.headers.get("X-CodyWatcher-Key")
       - Returns true if it matches expectedKey
       - Called in POST /notify and POST /shutdown handlers before processing
       - If header missing or mismatched → return new Response("Unauthorized", { status: 401 })

    2. **writeDecisionToFifo(fifoPath: string, decision: object): boolean**
       - Check existsSync(fifoPath) — if false, log warning and return false
       - Check statSync(fifoPath).isFIFO() — if false, log warning and return false
       - Open FIFO for writing: openSync(fifoPath, "w")
       - Write JSON string: writeSync(fd, JSON.stringify(decision) + "\n")
       - Close: closeSync(fd)
       - Return true on success, false on ENOENT error

    Read the CODYWATCHER_KEY from process.env at startup. If not set, log a warning but do not fail (allow development without the key, but log that auth is disabled).

    Create daemon.test.ts with tests for:
    - checkAuthHeader returns true with matching key, false without
    - writeDecisionToFifo returns false for non-existent path
    - writeDecisionToFifo returns false for regular file (not FIFO)

    Use bun test. Mock file system operations where needed.
